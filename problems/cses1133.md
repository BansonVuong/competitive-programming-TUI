Key insight: use 2 dfs to find the distance of everything.

first DFS:
DFS to find # of nodes in the subtree of x including x.
Basically, dfs, and then compute on the way up (after the dfs call to child nodes)

This returns 3 things: sz[x], which is the size of the subtree of x, including itself
down[x], which is the sum of the distance from x to all of its descendants
and ans[1], which is the sum of distance from 1 to all descendants

sz[par] = sum(sz[child]) + 1;
down[par] = sum(sz[child] + down[child]) 
why sz[child] + down[child]? because we must add 1 for each additional distance (each sz) to the down value of each child.

then, dfs2 just allows us to quickly compute the answer for each node.

we can subtract 1 for each sz from the distance of the direct parent of the node, because it is now 1 closer to all of its descendants than its parent, and then add 1 for each n that is not in that child's subtree

so 
```cpp
ans[child] = 
    ans[par] // parent's distance
    - sz[child] // 1 closer to child's children
    + (n-sz[child]) // 1 further from all other nodes
```
